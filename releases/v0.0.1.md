# camt053-loader-mt940 v0.0.1

Initial release. SWIFT MT940 → camt.053 `ParsedDocument` loader.

## What's in the box

A single public function:

```python
from camt053_loader_mt940 import parse_mt940

document = parse_mt940(mt940_text)
```

Returns a `camt053.models.ParsedDocument` with the same shape as the
camt.053 XML parser, so every downstream consumer in the suite (the
writer, validator, reversal builder, MCP and LSP servers) works
without further changes.

## Supported MT940 fields

| Tag | Mapped to |
| :--- | :--- |
| `:20:` | `ParsedDocument.msg_id` |
| `:25:` | `Statement.account` (IBAN or proprietary + optional BIC) |
| `:28C:` | `Statement.id` + `Statement.electronic_seq_nb` |
| `:60F:` / `:60M:` | `Balance` with `type_code="OPBD"` |
| `:61:` | `Entry` (amount, dc, dates, bank ref, customer ref) |
| `:86:` | `TransactionDetails.additional_info` on the preceding entry |
| `:62F:` / `:62M:` | `Balance` with `type_code="CLBD"` |
| `:64:` | `Balance` with `type_code="CLAV"` |
| `:65:` | `Balance` with `type_code="FWAV"` |

Reversal detection: `:61:` lines with debit/credit code `RD` or `RC`
map to entries with `reversal_indicator=True`.

## Why this exists

SWIFT MT940 is officially retiring in **November 2028**. This loader
bridges the two-year window where banks still produce MT940 but
downstream tooling expects camt.053.
